# breath-synth

A procedural breathing synthesiser for macOS. It assembles **exact-duration** inhale/exhale
breaths (1–30s) from a small palette of breath assets, using attack → sustain → release with
equal-power crossfade looping and a natural amplitude envelope — so any duration sounds
organic instead of like a single clip stretched to fit.

- **Asset-driven.** A small breath palette is generated once with ElevenLabs (start / loopable
  sustain / end clips per breath type and style). At runtime the engine loops, crossfades, and
  envelopes those clips to hit whatever duration you ask for. It makes no sound until a palette
  exists (run `generate-assets` or `dev-stub-assets` first).
- **Native.** Swift + AVFoundation. No web stack.
- **Library + CLI.** `BreathEngine` is a reusable Swift package target; `breath` is a CLI harness
  to generate assets, render to WAV, play single breaths, and loop full breathing cycles.

## Requirements

- macOS 13+
- A recent Swift toolchain (built with Swift 6.3, swift-tools-version 6.3)

## Build & test

```sh
swift build
swift test          # pure-DSP unit tests (no audio hardware needed)
```

## Quick start (no API key)

Generate synthetic placeholder assets so you can try the whole pipeline offline:

```sh
swift run breath dev-stub-assets --styles neutral,calm
swift run breath play --type inhale --duration 12 --style neutral
swift run breath cycle --inhale 4 --hold-in 4 --exhale 8 --hold-out 0 --loop
```

`dev-stub-assets` synthesizes filtered-noise breaths purely for development. They are *not*
production quality — use `generate-assets` for the real palette.

## Generating the real palette (ElevenLabs)

The engine consumes assets generated by the [ElevenLabs Sound Effects API](https://elevenlabs.io/docs/api-reference/text-to-sound-effects/convert).
Provide your key via the environment (no `.env` file is read or written):

```sh
export ELEVENLABS_API_KEY=sk_...
swift run breath generate-assets --styles neutral,calm
```

This writes WAV clips plus `manifest.json` into `Assets/breaths/`. Loopable sustain textures are
requested with the API's `loop` flag; clips are fetched as PCM and wrapped as WAV. If your account
tier rejects `pcm_44100`, pass a lower rate, e.g. `--output-format pcm_24000` (assets are resampled
to the 44.1 kHz working rate on load):

```sh
swift run breath generate-assets --styles neutral --output-format pcm_24000
```

Generated breaths are raw and may benefit from a light manual pass (e.g. in Audacity); the engine's
crossfade also masks minor loop seams.

## CLI reference

| Command | What it does |
| --- | --- |
| `generate-assets [--styles a,b] [--output-format pcm_44100] [--force]` | Generate the palette via ElevenLabs + write `manifest.json`. |
| `dev-stub-assets [--styles a,b]` | Synthesize offline placeholder assets for testing. |
| `play --type inhale --duration 12 --style calm [--seed N] [--no-variation]` | Render and play one breath. |
| `cycle --inhale 4 --hold-in 4 --exhale 8 --hold-out 0 [--loop] [--cycles N] [--style s]` | Play a breathing cycle. |
| `render --type exhale --duration 8 --out out.wav [--style s] [--seed N]` | Render one breath to a WAV file. |

All commands accept `--assets <dir>` (default `Assets/breaths`).

## How it works

```
generateBreath(type, duration):
  duration < 1.5s  → resample a one-shot / loop window to the exact length
  else             → start clip ──┐
                     loopable sustain, equal-power crossfaded to fill the middle
                     end clip   ──┘  all joined with equal-power crossfades
  then apply the natural amplitude envelope (endpoints forced to 0 → click-free)
  + subtle seeded variation (gain ±2 dB, loop playback-rate ±2%)
```

Each breath is rendered to an `AVAudioPCMBuffer` (cached, sample-exact, reproducible per seed) and
played via `AVAudioEngine`. Cycles concatenate inhale + silence + exhale + silence and loop the
composite buffer gaplessly.

### Architecture

- `Sources/BreathEngine/DSP/` — pure, AVFoundation-free, unit-tested math: envelopes, equal-power
  crossfades, segment/loop geometry, resampling, seeded variation.
- `Sources/BreathEngine/Assembly/` — `BreathAssembler` (pure `[Float]` assembly) + `AssetLibrary`
  (decode/downmix/resample assets to mono float).
- `Sources/BreathEngine/Playback/` + `BreathEngine.swift` — AVAudioEngine playback and the facade.
- `Sources/BreathCLI/` — the `breath` command and ElevenLabs generation.
